# moonzip 설계 문서

**작성일:** 2026-03-15
**도메인:** moonzip.kr
**한 줄 설명:** 고등 수학 문제집 전문 검색 & 난이도 투표 사이트

---

## 1. 배경 & 목적

개별과외 수학 강사가 학생별 커리큘럼을 짤 때, 교보문고/알라딘은 문제집 특화 필터가 없고 블로그/유튜브 정보는 주관적이고 부족함. 교보문고 스크래핑으로 수집한 고등 수학 문제집 DB를 기반으로, 강사·학생·학부모의 집단지성(난이도 투표 + 리뷰)을 활용한 검색 서비스를 만든다.

**타겟 유저:** 개별과외 수학 강사, 학생, 학부모

---

## 2. 기술 스택

| 항목 | 선택 |
|------|------|
| 프론트엔드 | Next.js 16 + React 19 + Tailwind v4 |
| 데이터베이스 | Supabase (PostgreSQL) |
| 배포 | Vercel |
| 도메인 | moonzip.kr (가비아) |

---

## 3. 데이터 파이프라인

```
교보문고 스크래핑 (Python)
        ↓
   Excel/XLSX (book-list/db/)
        ↓  import 스크립트
   Supabase DB
        ↓
Next.js API Routes
        ↓
   React 프론트엔드
        ↓
   Vercel → moonzip.kr
```

## 4. DB 스키마

### series — 시리즈 마스터 (핵심 단위)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| name | text UNIQUE | 시리즈명 (예: 쎈, 블랙라벨) |
| slug | text UNIQUE | URL용 슬러그 (영문/숫자/하이픈, 예: ssen, black-label) |
| types | text[] | 유형 배열 (개념서/유형서/심화서/기출문제집/N제/EBS) |
| created_at | timestamp | |

> slug 규칙: 한글 시리즈명을 로마자 발음 기반으로 수동 지정 또는 자동 생성 후 관리자 검토. URL은 `/series/[slug]` 형태 사용.

### books — 문제집 (교보문고 데이터)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | text PK | 판매상품ID |
| isbn | text | 상품코드 |
| title | text | 상품명 |
| series_id | uuid | → series.id |
| subject | text | 과목 (상품명 기반, 원본 그대로) |
| author | text | 저자 |
| publisher | text | 출판사 |
| released_at | date | 출시일 |
| edition | text | 개정판여부 (최신판/구판) |
| status | text | 판매현황 |
| price | integer | 판매가 |
| link | text | 교보문고 링크 |
| cover_image | text | 표지이미지 URL |

### users — 회원

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | Supabase Auth UID |
| email | text | 이메일 |
| role | text | 강사 / 학생 / 학부모 |
| created_at | timestamp | 가입일 |

> 추후: 강사 인증 뱃지 (is_verified_instructor boolean) 추가 예정

### reviews — 리뷰 (난이도 투표 포함)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | |
| series_id | uuid | → series.id |
| user_id | uuid | → users.id |
| difficulty | integer | 난이도 1~10 (필수) |
| content | text | 한줄 리뷰 (필수) |
| exam_type | text | 내신 / 정시 (선택) |
| grade | text | 1등급 / 2등급 / 3등급 / 4등급 / 5등급 / 6이하 (선택) |
| created_at | timestamp | |

> UNIQUE 제약: (user_id, series_id) — 1인 1시리즈 1리뷰, 수정은 upsert로 처리
> 난이도 평균은 reviews.difficulty 집계로 실시간 계산 (별도 테이블 불필요)

## 5. 페이지 구성

| 경로 | 설명 |
|------|------|
| `/` | 메인 (검색 & 필터) |
| `/series/[slug]` | 시리즈 상세 (문제집 목록 + 리뷰) |

## 6. 메인 페이지 UI

```
[유형 복수선택 ▼]  [시리즈명 검색______________🔍]

난이도  ●──────────────────────●  1 ~ 10
        ▲          ▲              ▲
      수력충전     쎈           블랙라벨

─────────────────────────────────────────────────
쎈            유형서          난이도 6.2  (142명 투표)
              [공통수학1] [공통수학2] [미적분1] [확률과통계]

개념원리       개념서          난이도 3.8  (89명 투표)
              [공통수학1] [공통수학2] [미적분1] [기하]

블랙라벨       심화서          난이도 8.9  (203명 투표)
              [공통수학1] [미적분1] [확률과통계]

규토 N제       N제             난이도 5.0  ✦ 아직 투표 없음
              [수능]
─────────────────────────────────────────────────
```

- 검색 결과는 **시리즈 단위**로 표시
- 투표 없으면 난이도 기본값 5.0 + "아직 투표 없음" 표시 → 첫 투표 유도
- 슬라이더 마커: 구간별(1~3, 4~7, 8~10) 최다 리뷰 시리즈를 실시간으로 표시. 초기값은 운영자가 직접 투표(수력충전 3점, 쎈 6점, 블랙라벨 8점)로 시드 데이터 확보

## 8. 리뷰 작성 UX

- 별도 페이지 이동 없이 **인라인 모달**로 작성
- 난이도: 큰 숫자 버튼 탭 한 번 (1~10)
- 텍스트: 작은 입력창, placeholder "한 줄이면 충분해요"
- 추천 대상: 맨 아래 선택사항 명시 (내신/정시, 등급)
- 제출 즉시 내 리뷰 반영 (낙관적 업데이트)
- 첫 투표 시 "첫 번째 리뷰어예요! 🎉" 피드백

## 9. 회원 시스템

- 리뷰에 역할 자동 표시
- 추후: 강사 인증 뱃지 (별도 인증 프로세스)

## 10. 콜드스타트 전략

- 투표 0개 → 난이도 5.0 기본값 + "아직 투표 없음" 표시
- "첫 번째가 되세요!" UX로 초기 참여 유도
- 서비스 초기에는 운영자(강사 본인)가 주요 시리즈 리뷰 직접 작성

## 11. MVP 범위

**포함:**
- 문제집 검색 & 필터 (유형, 키워드, 난이도 범위)
- 시리즈 카드에 과목 태그 **표시** (필터 기능은 미포함)
- 시리즈 상세 페이지 (문제집 목록 + 교보문고 링크)
- 회원가입 / 로그인
- 리뷰 & 난이도 투표
- 리뷰 수정 / 삭제 (본인)
- 마이페이지: 내 리뷰 목록 (최신순 / 난이도순 정렬)

**미포함 (추후):**
- 강사 인증 뱃지
- 과목별 필터 (메인 검색)
- 타 플랫폼 연동 (ISBN 활용)
- 추천 알고리즘

## 12. 운영 & 관리

- 관리자 기능은 별도 어드민 페이지 없이 **Supabase 대시보드**로 직접 처리
- 부적절한 리뷰 삭제, books 데이터 수정 등 모두 대시보드에서 수행
- 콜드스타트: 운영자(강사 본인)가 주요 시리즈 리뷰 직접 작성으로 초기 데이터 확보

## 13. 기타 결정 사항

- **난이도 슬라이더 & 투표 없는 시리즈**: 기본값 5.0으로 필터에 포함. "아직 투표 없음" 표시로 구분
- **리뷰의 강사 표시**: role='강사' 자동 표시. MVP에서 인증 뱃지(✓)는 미사용, 단순 텍스트로 표시
- **교보문고 데이터 갱신**: 초기 import 후 Python 파이프라인으로 주기적(주 1회) 수동 실행
